# Visual Rhyme · Website Image Brief
### For Fooocus generation · SDXL · brand-aligned premium photography

**Brand palette:** Deep purple `#9D20D6` · Violet `#7C3AED` · Lavender `#C084FC` · Near-black `#0A0010` · Sparingly: white highlights
**Aesthetic target:** Cinematic photographic realism. Samsung "The Wall" × Apple "Vision Pro" launch energy. No anime, no cartoon, no stock-photo look.
**Universal Fooocus settings:**
- **Performance:** Quality (or Speed for first pass)
- **Style:** `Fooocus Enhance` + `Fooocus Photograph` + `Fooocus Sharp`
- **Sampler:** dpmpp_2m_sde_gpu / karras
- **Output:** PNG, 1.5× upscale via "Quality" tab

**Universal negative prompt** (paste into every generation):
> `blurry, low quality, low resolution, watermark, signature, text overlay, ugly, deformed, oversaturated, cartoon, anime, illustration, render artifact, fake, stock photo, lens flare overuse, harsh shadows, plastic skin, distorted faces`

**Output location:** Save all to `/public/images/` with the exact filenames listed below. Then refresh the dev server — the site will pick them up automatically.

## INTEGRATION (already wired by Jarvis)

The site has `<img>` slots pre-configured with `onError` fallbacks pointing to `/images/<filename>`. As soon as you drop a generated file into `D:\visual-rhyme-website\public\images\` with the matching name, refresh the dev server (`localhost:5173`) — the image will appear. No code change needed.

Until images are dropped in, the existing gradient/3D fallbacks render — so the site never breaks.
